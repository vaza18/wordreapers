# Заява про використання штучного інтелекту та відповідальність

**Wordreapers** (Словозбирачі)

**Автор проєкту:** Василь Зайцев

Ця заява опублікована для **прозорості перед гравцями** додатку Wordreapers / Словозбирачі. Дата «останнє оновлення» змінюється лише при суттєвій зміні цієї політики або переліку інструментів ШІ.

**Останнє оновлення:** липень 2026

## 1. Які системи штучного інтелекту використовувались

Під час розробки цього проєкту я користувався такими інструментами на основі штучного інтелекту:

| Інструмент                        | Постачальник | Призначення                                                                                              |
| --------------------------------- | ------------ | -------------------------------------------------------------------------------------------------------- |
| **Claude** (Sonnet 4.5, Opus 4.6) | Anthropic    | Генерація та редагування коду, документації, тестів, скриптів; пояснення технічних рішень                |
| **Cursor IDE**                    | Anysphere    | Середовище розробки з інтегрованими AI-асистентами для написання, рефакторингу та пошуку по кодовій базі |
| **ChatGPT**                       | OpenAI       | Генерація іконки та логотипу додатку                                                                     |

Ці інструменти використовувались **виключно на етапі розробки**. Вони **не входять до складу** мобільного додатку Wordreapers / Словозбирачі і **не обробляють** ігрові дані гравців під час гри.

**Що не використовується в продукті для гравців:**

- немає чат-ботів, генеративних підказок чи «розумних» суперників на основі LLM;
- перевірка слів відбувається **локально** за вбудованим словником [ВЕСУМ](https://github.com/brown-uk/dict_uk), без звернень до сторонніх AI-сервісів;
- мультиплеєр працює через хмарну інфраструктуру Google Firebase (синхронізація сесій, автентифікація), а не через мовні моделі.

## 2. Як штучний інтелект зробив внесок у проєкт

ШІ допомагав як **інструмент продуктивності розробника**, аналогічно компілятору, IDE або статичному аналізатору. Типовий внесок:

- **Код додатку:** екрани інтерфейсу (Expo / React Native), ігрова логіка, онлайн-синхронізація, тести.
- **Інфраструктура:** скрипти збірки словника з ВЕСУМ, CI-перевірки, конфігурація Firebase, генерація звуків зворотного зв'язку.
- **Документація:** правила гри, юридичні тексти, README, план реалізації.
- **Графіка:** іконка та логотип додатку (ChatGPT) — не генеруються під час гри.

**Що залишилось за мною як автором:**

- ідея гри (сімейна традиція, адаптація паперової гри в додаток);
- продуктові рішення: правила, UX, обмеження публічних кімнат, політика мов;
- прийняття або відхилення пропозицій ШІ; фінальний вибір архітектури та поведінки додатку;
- ручна перевірка вибірки словника;
- сімейне тестування гри (playtest) перед і після релізу;
- публікація, підтримка репозиторію та відповідальність перед користувачами.

Критичні рішення та прийняття результату **завжди за людиною**, а не за моделлю.

## 3. Процес перевірки та контролю якості

Щоб зменшити ризик помилок від автоматично згенерованого коду, застосовується такий процес:

1. **Автоматичні перевірки перед злиттям коду** (`npm run ci:check`):
   - ESLint (стиль і типові помилки);
   - Prettier (форматування);
   - TypeScript typecheck;
   - Vitest — понад 125 unit-тестів (ігрова логіка, словник, онлайн-сценарії, UI-допоміжні функції).
2. **CI на GitHub** — на pull request запускаються лише релевантні перевірки за зміненими файлами; повний набір перевірок — на кожному push/merge до `main`. Прямі push до `dev` CI не запускають (локально: `npm run ci:check`).
3. **Pre-commit hook (Husky)** — локальний запуск `ci:check` перед комітом.
4. **Словник:** окремі скрипти `dict:validate` (регресійні тест-кейси) та ручний огляд вибірки `dict:review-sample`.
5. **Безпека онлайн-гри:** тести правил Firebase Realtime Database, App Check у production-збірках.
6. **Ручне тестування:** гра з родиною на реальних пристроях; перевірка сценаріїв тренування, мультиплеєра, паузи, результатів.
7. **Прозорість:** вихідний код [відкритий на GitHub](https://github.com/vaza18/wordreapers) під ліцензією MIT — будь-хто може перевірити поведінку додатку, зокрема щодо даних і перевірки слів.

Жоден фрагмент коду не потрапляє до релізу лише тому, що його згенерував ШІ: він проходить описані перевірки та мою оцінку.
